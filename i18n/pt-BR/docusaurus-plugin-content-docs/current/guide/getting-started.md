# Primeiros Passos

## Criando um Novo Projeto com os Templates do Serenity BDD

A maneira mais rápida de começar com um projeto Serenity BDD é usar um dos projetos iniciais. Estes são projetos de template disponíveis no Github, que contêm uma estrutura básica de projeto Serenity BDD e uma implementação de teste única. Estes projetos de template são mantidos atualizados com a versão estável mais recente do Serenity BDD.

Os principais projetos iniciais disponíveis são os seguintes:

| Template | Descrição | Começar |
| -------- | ----------- | ------------|
| [Serenity BDD com JUnit](https://github.com/serenity-bdd/serenity-junit-starter) | Uma estrutura de projeto simples usando Serenity BDD, JUnit 5 e uma arquitetura de Action Class/Page Object. | **[Usar Este Template](https://github.com/serenity-bdd/serenity-junit-starter/generate)** |
| [Serenity BDD com JUnit e Screenplay](https://github.com/serenity-bdd/serenity-junit-screenplay-starter) | Um projeto construído em torno do Screenplay Pattern usando Serenity BDD, JUnit 5 e Serenity Screenplay |  **[Usar Este Template](https://github.com/serenity-bdd/serenity-junit-screenplay-starter/generate)** |
| [Serenity BDD com Cucumber e Screenplay](https://github.com/serenity-bdd/serenity-cucumber-starter) | Uma suíte de testes no estilo BDD usando Cucumber e Serenity Screenplay | **[Usar Este Template](https://github.com/serenity-bdd/serenity-cucumber-starter/generate)** |

Para criar um novo projeto, vá até o template de projeto que você deseja usar e clique no botão "Use this template". Isso permitirá que você crie um novo projeto no seu próprio repositório Github.

![](img/use-this-template.png)

Se você está trabalhando em um projeto interno, você também pode baixar o projeto de template clicando no botão "Code".
